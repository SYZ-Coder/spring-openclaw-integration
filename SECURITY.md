# Security Policy

## Supported Versions

Security fixes are generally applied to the latest maintained version on the default branch.

## Reporting a Vulnerability

Please do not report security issues through public GitHub issues.

Instead, contact the maintainers privately and include:

- A clear description of the issue
- Steps to reproduce it
- Impact assessment
- Suggested mitigation, if known

If no dedicated security mailbox exists yet, repository owners should add one before broad public promotion of the project.

## Response Expectations

- We will acknowledge a valid report as soon as practical.
- We will evaluate severity and impact before public disclosure.
- We may ask for additional reproduction details.
- We aim to coordinate a fix before publishing details.

## Sensitive Data

Never include real tokens, secrets, customer data, or production endpoints in reports unless explicitly requested through a secure channel.
